# 开篇

## 为什么写博客

建这个站的时候我问了自己一句：网上已经那么多技术文章了，多写一篇又有什么意义？

后来想通了——**写博客不是为了填补互联网的信息缺口，而是为了填补自己的理解缺口。**

一个知识点，你看懂了是一回事，能讲清楚是另一回事。写下来的过程中，你会被迫面对那些「以为自己懂了但其实没懂」的角落。所以这首先是一份**写给自己看的笔记**，如果恰好也帮到了别人，那是 bonus。

## 这里会有什么

目前打算围绕几个方向来写：

- **后端踩坑** — Go、Rust、分布式系统，实际项目中碰到的问题和溯源过程
- **源码阅读** — 读一些优秀开源项目的实现，记录设计思路
- **工程实践** — CI/CD、可观测性、性能调优，折腾过的事
- **偶尔感想** — 关于技术选型、职业成长之类的碎碎念

不会追求高频更新，但会尽量保证**每篇都有扎实的内容**。

## 这个站怎么搭的

简单说一下技术栈，也算留个记录：

- **[VitePress](https://vitepress.dev)** — 静态站点生成，Markdown 直接写，构建飞快
- **GitHub Actions** — push 即部署，rsync 推到服务器
- **Nginx** — 挂个静态目录，配好 SSL 就完事

整套下来零运行时依赖，维护成本趋近于零。后面可能会加评论系统和站内搜索，但目前的极简状态用着挺舒服。

## 最后

如果你通过某篇文章来到这个站，欢迎通过邮件或 GitHub 联系我。

希望一年后回头看，这里能积累一些拿得出手的东西。
